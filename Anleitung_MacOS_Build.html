🎬 Slideshow Generator

Schritt-für-Schritt-Anleitung: App für macOS erstellen

Für macOS · Ohne Programmierkenntnisse

Überblick – Was passiert hier?

Diese Anleitung erklärt, wie man aus dem Diashow Generator eine eigenständige macOS-App erstellt – eine Datei mit der Endung .app, die man einfach per Doppelklick startet, ohne dass Python oder andere Programme installiert sein müssen.

ℹ️ Wichtig vorab Der Build-Prozess muss einmalig auf einem Mac durchgeführt werden. Das fertige SlideshowGenerator.app läuft dann auf jedem Mac und benötigt keine weitere Installation.
⏱ Zeitaufwand Ca. 30–45 Minuten beim ersten Mal (je nach Internetgeschwindigkeit). Danach dauert ein erneuter Build nur noch ~5 Minuten.

Voraussetzungen

  • Ein Mac (iMac, MacBook, Mac Mini) mit macOS 11 oder neuer
  • Internetverbindung (nur für die Einrichtung nötig)
  • Ca. 1 GB freier Speicherplatz
  • Den Ordner „Diashow Generator" (dieser Ordner, in dem diese Anleitung liegt)
ℹ️ Apple Silicon oder Intel? Die Anleitung funktioniert auf beiden Chip-Typen (M1, M2, M3, M4 sowie ältere Intel-Macs) ohne Unterschied.

Schritt-für-Schritt-Anleitung

1

Den Ordner auf den Mac kopieren

Kopiere den gesamten Ordner „Diashow Generator" auf den Mac, z.B. auf den Schreibtisch oder in den Dokumente-Ordner.

Das geht am einfachsten per USB-Stick, AirDrop oder OneDrive/iCloud.

✓ Ziel Der Ordner „Diashow Generator" liegt auf dem Mac und enthält die Dateien create_slideshow.py, slideshow_gui.py, requirements.txt usw.
2

Python installieren

Python ist die Programmiersprache, die der Slideshow Generator verwendet. Auf Macs ist oft eine veraltete Version vorinstalliert – wir brauchen Version 3.10 oder neuer.

So prüfen, ob Python schon installiert ist:

  1. Öffne das Terminal (siehe Schritt 3, wie das geht)
  2. Tippe folgendes ein und drücke Enter:
python3 --version

Wenn eine Zahl wie Python 3.11.4 erscheint und die Zahl nach dem Punkt ≥ 10 ist – super, Python ist bereits installiert, weiter zu Schritt 3.

Erscheint eine Fehlermeldung oder eine Zahl kleiner als 3.10:

Python herunterladen und installieren:

  1. Öffne den Browser und gehe auf python.org/downloads
  2. Klicke auf den großen gelben Button Download Python 3.x.x (die neueste Version wird automatisch vorgeschlagen)
  3. Die heruntergeladene Datei heißt z.B. python-3.13.0-macos11.pkg
  4. Doppelklick auf die Datei → dem Installations-Assistenten folgen (immer „Weiter" klicken)
  5. Am Ende auf „Zertifikate installieren" klicken (ein Doppelklick auf die Datei Install Certificates.command im Python-Ordner unter Programme)
✓ Prüfen Terminal öffnen → python3 --version eintippen → es sollte jetzt Python 3.10 oder höher erscheinen.
3

Das Terminal öffnen

Das Terminal ist das „Befehlsfenster" des Macs – ähnlich wie die Eingabeaufforderung (cmd) unter Windows. Hier gibt man Befehle ein.

Methode A – Spotlight (empfohlen):

  1. Drücke gleichzeitig ⌘ Command + Leertaste
  2. Im erscheinenden Suchfeld Terminal eintippen
  3. Auf „Terminal" in den Ergebnissen doppelklicken

Methode B – Finder:

  1. Finder öffnen
  2. Programme → Dienstprogramme → Terminal
  3. Doppelklick auf „Terminal"

Das Terminal sieht wie ein schwarzes oder weißes Fenster mit Text aus. Am Ende der Zeile blinkt ein Cursor – das ist normal.

4

In den „Diashow Generator"-Ordner wechseln

Damit alle folgenden Befehle im richtigen Ordner ausgeführt werden, muss man dem Terminal zuerst sagen, wo der Ordner liegt.

Der einfachste Weg:

  1. Tippe im Terminal cd (mit einem Leerzeichen danach, aber noch nicht Enter drücken)
  2. Öffne den Finder und navigiere zum Ordner „Diashow Generator"
  3. Ziehe den Ordner „Diashow Generator" mit der Maus direkt ins Terminal-Fenster
  4. Der Pfad wird automatisch eingetragen, z.B. cd /Users/anna/Desktop/Diashow\ Generator
  5. Drücke Enter
ℹ️ Alternative ohne Drag & Drop: Liegt der Ordner auf dem Schreibtisch, kann man direkt eintippen:

cd ~/Desktop/Diashow\ Generator

Das ~ steht für den persönlichen Ordner, \ vor dem Leerzeichen im Ordnernamen ist wichtig.
✓ Prüfen Nach dem Drücken von Enter sollte der Pfad im Terminal sichtbar sein, z.B. anna@MacBook Diashow Generator %. Tippe dann ls und Enter – es sollten Dateien wie create_slideshow.py und requirements.txt erscheinen.
5

Virtuelle Python-Umgebung erstellen

Eine „virtuelle Umgebung" ist ein isolierter Bereich, in dem alle benötigten Zusatzprogramme installiert werden – ohne das restliche System zu verändern. Das ist wie ein eigener Ordner nur für dieses Projekt.

⚠️ Wichtig Die .venv-Umgebung aus Windows funktioniert auf dem Mac nicht. Wir erstellen eine neue, macOS-kompatible Umgebung.

Tippe folgenden Befehl ein und drücke Enter:

python3 -m venv .venv-mac

Es passiert kurz etwas (keine Ausgabe ist normal), danach erscheint wieder der Cursor. Im Ordner wurde jetzt ein Unterordner .venv-mac angelegt.

Aktiviere die Umgebung mit diesem Befehl:

source .venv-mac/bin/activate
✓ Prüfen Am Anfang der Terminal-Zeile erscheint jetzt (.venv-mac) – die Umgebung ist aktiv. Beispiel: (.venv-mac) anna@MacBook Diashow Generator %
ℹ️ Hinweis Bei jedem neuen Terminal-Fenster muss die Umgebung erneut aktiviert werden (wieder source .venv-mac/bin/activate eingeben). Solange (.venv-mac) sichtbar ist, ist sie aktiv.
6

Benötigte Programme installieren

Jetzt werden alle Zusatzprogramme (sogenannte „Pakete") installiert, die der Slideshow Generator benötigt. Die Liste steht in der Datei requirements.txt.

Tippe ein und drücke Enter:

pip install -r requirements.txt

Jetzt werden automatisch mehrere Pakete heruntergeladen und installiert. Das kann 2–10 Minuten dauern, je nach Internetgeschwindigkeit. Es erscheinen viele Zeilen mit Text – das ist normal.

✓ Prüfen Am Ende erscheint eine Zeile wie Successfully installed imageio ... ohne rot hinterlegte Fehlermeldungen.
🚨 Falls ein Fehler erscheint Erscheint etwas wie ERROR: Could not build wheels oder command not found, prüfe ob (.venv-mac) am Zeilenanfang steht (Schritt 5 nochmals durchführen). Tritt der Fehler weiter auf, am Ende dieser Anleitung gibt es Hilfe-Hinweise.
7

PyInstaller installieren

PyInstaller ist das Werkzeug, das den Python-Code in eine eigenständige App verwandelt.

Tippe ein und drücke Enter:

pip install pyinstaller

Die Installation dauert ca. 1–2 Minuten.

✓ Prüfen Am Ende erscheint Successfully installed pyinstaller-...
8

Die App erstellen (Build starten)

Jetzt wird die eigentliche App gebaut. Tippe ein und drücke Enter:

python -m PyInstaller slideshow_generator.spec

Jetzt läuft der Build-Prozess. Es erscheinen viele Zeilen mit Text – das ist normal und zeigt den Fortschritt. Der Prozess dauert ca. 3–8 Minuten.

Warte, bis der Cursor wieder erscheint und keine neuen Zeilen mehr kommen.

✓ Prüfen – Erfolg Am Ende erscheint eine Zeile wie:
INFO: Build complete! The results are available in: .../dist
🚨 Falls Fehler erscheinen Rot hinterlegte ERROR-Zeilen bedeuten, dass etwas schiefgelaufen ist. Häufigste Ursache: die virtuelle Umgebung ist nicht aktiv. Führe nochmals source .venv-mac/bin/activate aus und wiederhole diesen Schritt.
9

Die fertige App finden und starten

Nach erfolgreichem Build liegt die App im Unterordner dist:

Diashow Generator → dist → SlideshowGenerator.app

Öffne diesen Ordner im Finder: Tippe im Terminal

open dist

Der Finder öffnet sich und zeigt die Datei SlideshowGenerator.app.

Diese .app-Datei kann man:

  • in den Programme-Ordner ziehen (für dauerhaften Zugriff)
  • ins Dock ziehen
  • direkt per Doppelklick starten
10

macOS-Sicherheitswarnung umgehen (einmalig)

Beim ersten Start erscheint möglicherweise eine Warnung: „SlideshowGenerator.app kann nicht geöffnet werden, weil es von einem nicht verifizierten Entwickler stammt."

Das ist normal für selbst erstellte Apps. So umgeht man die Warnung:

  1. Rechtsklick (oder Control + Klick) auf SlideshowGenerator.app
  2. Im Menü „Öffnen" auswählen (nicht Doppelklick!)
  3. Im Warndialog auf „Öffnen" klicken
ℹ️ Nur einmalig nötig Nach dem ersten Öffnen per Rechtsklick → „Öffnen" kann die App danach normal per Doppelklick gestartet werden.

Alternative (falls Rechtsklick nicht klappt):

  1. Apple-Menü → Systemeinstellungen
  2. Datenschutz & Sicherheit
  3. Ganz unten erscheint ein Hinweis zu SlideshowGenerator
  4. Auf „Trotzdem öffnen" klicken

Die App weitergeben

Die fertige SlideshowGenerator.app kann an andere Mac-Nutzer weitergegeben werden – sie benötigen kein Python oder andere Programme.

  • Per AirDrop, USB-Stick, iCloud oder E-Mail verschicken
  • Empfänger müssen ebenfalls einmalig per Rechtsklick → „Öffnen" starten
  • Funktioniert auf macOS 11 (Big Sur) und neuer
  • Funktioniert sowohl auf Intel- als auch Apple Silicon-Macs (sofern der Build auf demselben Chip-Typ durchgeführt wurde)
⚠️ Intel vs. Apple Silicon Eine auf einem M1/M2/M3-Mac gebaute App läuft möglicherweise nicht auf einem alten Intel-Mac (und umgekehrt). Für maximale Kompatibilität auf einem Intel-Mac bauen, wenn die Ziel-Nutzer verschiedene Macs haben. Oder den Build auf beiden Typen durchführen.

Schnellübersicht aller Befehle

Alles auf einen Blick (Terminal-Befehle in der richtigen Reihenfolge):

# 1. In den Ordner wechseln (Pfad anpassen!) cd ~/Desktop/Diashow\ Generator # 2. Virtuelle Umgebung erstellen (nur einmalig) python3 -m venv .venv-mac # 3. Virtuelle Umgebung aktivieren (bei jedem neuen Terminal-Fenster) source .venv-mac/bin/activate # 4. Pakete installieren (nur einmalig) pip install -r requirements.txt # 5. PyInstaller installieren (nur einmalig) pip install pyinstaller # 6. App bauen python -m PyInstaller slideshow_generator.spec # 7. Ergebnis-Ordner öffnen open dist

Häufige Probleme und Lösungen

„python3: command not found"

Python ist nicht installiert oder nicht gefunden. Zurück zu Schritt 2 und Python von python.org installieren.

„pip: command not found"

Die virtuelle Umgebung ist nicht aktiv. source .venv-mac/bin/activate erneut ausführen und darauf achten, dass (.venv-mac) am Zeilenanfang erscheint.

„No such file or directory: slideshow_generator.spec"

Das Terminal ist nicht im richtigen Ordner. Schritt 4 wiederholen – mit ls prüfen, ob slideshow_generator.spec sichtbar ist.

Die App öffnet sich, aber es passiert nichts / sie stürzt sofort ab

Starte die App einmal direkt aus dem Terminal, um Fehlermeldungen zu sehen:

./dist/SlideshowGenerator.app/Contents/MacOS/SlideshowGenerator

Die angezeigte Fehlermeldung gibt Hinweis auf das Problem.

„SlideshowGenerator.app ist beschädigt"

macOS Gatekeeper blockiert die App. Im Terminal ausführen:

xattr -dr com.apple.quarantine dist/SlideshowGenerator.app

Danach sollte die App per Doppelklick startbar sein.

imageio-ffmpeg Fehler / ffmpeg nicht gefunden

Das ffmpeg-Binary ist möglicherweise nicht korrekt eingebunden. Führe folgendes aus, um ffmpeg manuell für die aktuelle Plattform herunterzuladen:

python3 -c "import imageio_ffmpeg; print(imageio_ffmpeg.get_ffmpeg_exe())"

Erscheint ein gültiger Pfad, ist ffmpeg vorhanden. Den Build dann erneut starten.
